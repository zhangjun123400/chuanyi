# UI设计师采薇对 v2.0 用户系统 PRD 的建议

**日期**: 2026-05-18  
**版本**: 针对 PRD v2.0 Part 1 — 用户系统（2026-05-18 docx 版）

---

## 总体评价

PRD 整体质量很高。核心认证流程（登录/刷新/登出/改密）和边界条件（并发登录、批量吊销、最后一个 admin 保护）都已完整覆盖。数据所有权矩阵和错误码体系清晰统一，四阶段实施计划合理。

以下从 UI/UE 和前端落地角度提 6 点补充建议，供多灵参考。

---

## 建议 1：管理员后台页面结构优化 —— 用 Tab 替代纵向堆叠

**现状**: PRD 8.4 将用户列表和操作日志纵向堆叠在同一页面。

**问题**: 当用户超过 20 个时，页面会拉得很长，操作日志被挤到首屏以下，需要滚动才能看到。两个模块的信息密度和关注频率不同（用户管理高频，操作日志低频），堆叠在一起降低了操作效率。

**建议**: 用水平 Tab 切换「用户管理」和「操作日志」两个视图，各自独立滚动。这样：

- 用户管理保持紧凑，首屏即可完整操作
- 操作日志不被挤压，有自己的展示空间
- 切换 Tab 时保留各自的筛选/分页状态

```
┌──────────────────────────────────────────────┐
│  TryOn Studio · 管理后台                     │
├──────────────────────────────────────────────┤
│  [ 用户管理 ]  [ 操作日志 ]                  │
│                                              │
│  （当前 Tab 的内容区域）                      │
└──────────────────────────────────────────────┘
```

## 建议 2：「记住我」增加用户可感知的提示

**现状**: PRD 只有 checkbox + 后端有效期逻辑，没有前端提示文案。

**问题**: 用户不知道勾选和不勾选的区别是什么，也不知道当前登录会保持多久。

**建议**:

- 勾选时旁边显示「30 天内无需重新登录」
- 未勾选时显示「7 天内保持登录」
- 登录成功后，在主应用 sidebar 用户区底部显示「登录有效期至 6月17日」，让用户感知剩余时间

```
┌────────────────────┐
│ 👤 运营管理员      │
│    admin · 1200 额度│
│                    │
│ 有效期至 6月17日    │
│ [修改密码] [登出]  │
└────────────────────┘
```

## 建议 3：修改密码后的过渡体验

**现状**: PRD 4.6 说「修改密码后清除所有令牌 → 跳回登录页」。

**问题**: 用户主动修改密码后，突然被踢回一个空的登录页，没有上下文，体验断裂。用户可能以为是系统出错。

**建议**: 修改密码成功后展示一个过渡提示页，告知发生了什么，然后引导用户重新登录：

```
┌──────────────────────────────────┐
│                                  │
│          ✓ 密码已修改            │
│                                  │
│   为保障账号安全，所有设备       │
│   已退出登录                    │
│                                  │
│   请使用新密码重新登录           │
│                                  │
│      3 秒后自动跳转登录页        │
│      [ 立即登录 ]               │
│                                  │
└──────────────────────────────────┘
```

## 建议 4：管理员后台免密进入加过渡提示

**现状**: PRD 4.4「如果主应用已登录且是 admin，后台可免密直接进入（共享 localStorage）」。

**问题**: 管理员在浏览器新标签页打开 localhost:3001，如果主应用已登录，会直接进入后台管理界面。这个过程对用户是「黑盒」—— 为什么打开新地址就直接进入了？缺少感知。

**建议**: 加一个轻量过渡页（约 1-2 秒），告知正在使用的身份信息，如果不符合预期可以退出：

```
┌──────────────────────────────────┐
│                                  │
│        TryOn Studio             │
│        管理后台                  │
│                                  │
│   检测到已登录的管理员账号：     │
│   admin@tryonstudio.local        │
│                                  │
│   正在进入管理后台...            │
│                                  │
│   [ 这不是我，退出登录 ]         │
│                                  │
└──────────────────────────────────┘
```

如果 2 秒内用户没有操作，自动跳转进入后台。这比直接进入更透明、更安全。

## 建议 5：补充空态、Loading 态、错误态的 UI 设计

**现状**: PRD 只覆盖了正常流程的界面描述。

**问题**: 以下状态在真实使用中必然出现，但没有设计：

| 场景 | 缺失的状态 |
|------|-----------|
| 新租户刚部署，无任何用户 | 用户列表空态 |
| 搜索无结果 | 搜索空态 |
| 用户列表加载中 | 骨架屏/Loading Spinner |
| 创建/编辑用户提交中 | 按钮 Loading + 防重复提交 |
| 分页数据加载中 | 分页区域 Loading 指示 |
| 网络请求失败 | 全局错误 Toast/提示 |
| 删除/禁用操作确认 | 二次确认弹窗样式 |

**建议**: 在设计稿中为以上状态补充独立的画板，确保研发团队有明确的 UI 参照。关键的空态设计原则：

- 空态不是「什么都没有」，而是「告诉用户这里可以做什么」
- 例如：用户列表为空时显示「暂无用户」+「创建第一个用户」引导按钮

## 建议 6：创建用户后密码展示的安全性体验

**现状**: PRD 只说「弹窗展示随机密码，提示复制保存」。

**问题**: 密码是敏感信息。直接明文展示有屏幕被旁人看到的安全风险，且没有提到复制交互细节。

**建议**:

- 密码默认脱敏显示（`a3f8••••`），旁边有 👁 点击查看完整密码的 toggle
- 提供「📋 一键复制」按钮，复制成功后按钮短暂变为绿色「✓ 已复制」
- 弹窗底部红色警告文案：「⚠️ 此密码仅展示一次，关闭弹窗后将无法找回。请妥善保存并交给用户」

```
┌──────────────────────────────────────┐
│  用户创建成功                        │
│                                      │
│  新用户：new_user@tryonstudio.local  │
│                                      │
│  初始密码：                          │
│  ┌──────────────────────────┐        │
│  │  a3f8••••      [👁]     │        │
│  └──────────────────────────┘        │
│  [ 📋 一键复制 ]                     │
│                                      │
│  ⚠️ 此密码仅展示一次，关闭弹窗后    │
│  将无法找回，请妥善保存并交给用户    │
│                                      │
│              [ 确认已保存 ]          │
└──────────────────────────────────────┘
```

按钮文案用「确认已保存」而非「关闭」，让管理员有明确的确认动作。

---

## 总结

以上 6 点建议都聚焦于 **PRD 已有流程的 UI/UE 补全**，不涉及新增功能或架构变更。优先级排序：

| 优先级 | 建议 | 原因 |
|--------|------|------|
| P0 | 建议 5：空态/Loading/错误态 | 直接影响研发落地，没有设计就无法实现 |
| P0 | 建议 6：密码展示安全性 | 涉及安全体验，首次使用就会遇到 |
| P1 | 建议 1：Tab 替代堆叠 | 影响后台操作效率，但页面结构可以后期调整 |
| P1 | 建议 3：修改密码过渡页 | 影响用户体感，但不阻塞功能上线 |
| P2 | 建议 2：记住我用户提示 | 锦上添花，核心功能不受影响 |
| P2 | 建议 4：后台免密过渡页 | 体验优化，但仅影响 admin 角色 |
