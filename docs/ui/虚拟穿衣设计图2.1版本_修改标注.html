P0 重置密码确认使用自定义弹窗替代原生 confirm() 修复项 #2

问题：管理员对用户执行「重置密码」操作时，当前使用浏览器原生 confirm() 对话框。原生弹窗在不同操作系统样式不一致，无品牌感，且无法自定义按钮文案和警告信息。
❌ 修改前（当前实现）
当前：浏览器原生 confirm() 弹窗

127.0.0.1:3001 显示：

确定要重置用户 xxx 的密码吗？此操作不可撤销。

取消 确定
⚠ 浏览器原生弹窗，macOS / Windows 样式不一致，无品牌感
⚠ 无法展示详细警告信息（如告知用户将被强制退出）
✅ 修改后（设计稿）
目标：自定义品牌确认弹窗
⚠️

重置用户密码

确定要重置用户 zhangsan@123.com 的密码吗？

⚠ 重置后该用户的所有登录会话将立即失效，需使用新密码重新登录。
✓ 品牌化设计，与整体 UI 风格统一
✓ 黄色警告区域告知影响范围（所有会话失效）
✓ 危险操作用红色按钮「确认重置」明确语义

P1 「记住我」增加动态提示文案 修复项 #3

问题：主应用登录页「记住我」复选框文案固定为「记住我（30天免登录）」，无论勾选与否都显示相同文字。用户无法感知勾选和不勾选的区别。管理员后台登录页的「记住我」则完全没有提示文案。
